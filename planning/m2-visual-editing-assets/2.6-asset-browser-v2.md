## Visão Geral

O Asset Browser v2 é uma evolução significativa da gestão de ficheiros dentro do Caffeine Studio. Enquanto a v1 servia apenas para listar ficheiros, a v2 introduz capacidades de visualização ricas, incluindo thumbnails gerados pela GPU para texturas, um painel de preview detalhado e ferramentas de organização avançadas como filtros e pesquisa.

Esta ferramenta é o centro nevrálgico para a interação com assets, fornecendo a interface de origem para o sistema de Drag-and-Drop e permitindo a gestão direta da hierarquia de ficheiros do projeto sem sair do editor.

---

## Implementação

O Asset Browser utiliza um sistema de caching para thumbnails e uma vista de grelha (Grid View) otimizada para lidar com centenas de assets.

### Estrutura de Classes

```cpp
namespace Caffeine::Editor {

    class AssetBrowser {
    public:
        void OnImGuiRender();
        void Refresh(); // Recarrega a árvore de diretórios

    private:
        void RenderToolbar();
        void RenderSidebar(); // Árvore de pastas
        void RenderContent(); // Grelha de assets
        void RenderPreviewPanel(); // Detalhes do asset selecionado

        void GenerateThumbnails();
        
        std::filesystem::path m_CurrentDirectory;
        std::string m_SearchFilter;
        float m_ThumbnailSize = 64.0f;
        
        // Cache de texturas para ícones
        std::map<AssetHandle, RHI::Texture*> m_Thumbnails;
        AssetHandle m_SelectedAsset = 0;
    };
}
```

### Thumbnail Generation

Para texturas, o motor carrega uma versão reduzida (downsampled) para a GPU para usar como ícone na grelha.

```cpp
void AssetBrowser::GenerateThumbnails() {
    for (auto& entry : std::filesystem::directory_iterator(m_CurrentDirectory)) {
        if (IsTexture(entry) && !m_Thumbnails.count(GetHandle(entry))) {
            m_Thumbnails[GetHandle(entry)] = m_AssetManager->LoadThumbnail(entry.path());
        }
    }
}
```

### Funcionalidades de UI
- **Grid View**: Ícones grandes com nomes por baixo.
- **List View**: Detalhes técnicos (tamanho, tipo, data).
- **Search Bar**: Filtra os itens visíveis por nome em tempo real.

## Decisões de Design

| Decisão | Justificativa |
|---------|---------------|
| GPU Thumbnails | Usar a GPU para renderizar os previews permite que o editor permaneça responsivo durante o scroll de grandes pastas de imagens. |
| Drag Preview | Mostrar o ícone do asset junto ao rato durante o drag (Drag-and-Drop) melhora o feedback visual. |
| Breadcrumbs Navigation | Permite navegar rapidamente de volta para pastas superiores na hierarquia sem usar a barra lateral. |

---

## Critério de Aceitação

- [ ] Texturas (PNG/JPG) aparecem com thumbnails visuais na grelha.
- [ ] O painel de Preview mostra informações detalhadas (resolução, formato) ao selecionar um asset.
- [ ] A barra de pesquisa filtra assets por nome instantaneamente.
- [ ] É possível alternar entre vista de grelha e vista de lista.
- [ ] Clicar com o botão direito permite criar novas pastas ("New Folder").
- [ ] Arrastar um asset mostra uma miniatura flutuante que segue o cursor.
