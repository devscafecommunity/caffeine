## Visão Geral

O Project Manager é o ponto de entrada do Caffeine Studio. Ele permite que os utilizadores criem novos projetos a partir de templates, abram projetos existentes e façam a gestão da lista de projetos recentes. Um "Project" no Caffeine é definido por um ficheiro `.caffeine` (JSON) que contém metadados sobre o motor, caminhos de assets e configurações de build.

Esta ferramenta garante que o ambiente de desenvolvimento seja configurado corretamente, criando a estrutura de pastas necessária (`assets/`, `scripts/`, `build/`) e inicializando as bases de dados de assets para o novo projeto.

---

## Implementação

O Project Manager funciona como uma janela modal ou um executável de "launcher" separado antes de carregar o editor principal.

### Interface Visual (ASCII)

```
┌─────────────────────────────────────────────┐
│       ☕ Caffeine Studio — Project Manager   │
├─────────────────────────────────────────────┤
│  Recent Projects:                           │
```

### Estrutura de Dados e Classes

```cpp
namespace Caffeine::Editor {

    struct ProjectConfig {
        std::string Name;
        std::string Version;
        std::filesystem::path RootPath;
        std::filesystem::path AssetPath;
        std::string TemplateType; // "2D", "3D", "Empty"
    };

    class ProjectManager {
    public:
        bool CreateNewProject(const ProjectConfig& config);
        bool OpenProject(const std::filesystem::path& path);
        
        const std::vector<std::filesystem::path>& GetRecentProjects() const;
        const ProjectConfig& GetCurrentProject() const { return m_CurrentConfig; }

    private:
        void CreateDirectoryStructure(const std::filesystem::path& root);
        void SaveProjectFile(const ProjectConfig& config);
        void UpdateRecentProjects(const std::filesystem::path& path);

        ProjectConfig m_CurrentConfig;
        std::vector<std::filesystem::path> m_RecentProjects;
    };
}
```

### Ficheiro de Configuração (`project.caffeine`)

```json
{
  "project_name": "MyGame",
  "engine_version": "0.2.0",
  "paths": {
    "assets_raw": "assets/raw",
    "assets_processed": "assets/processed",
    "scripts": "scripts"
  },
  "last_scene": "scenes/main.scene"
}
```

## Decisões de Design

| Decisão | Justificativa |
|---------|---------------|
| JSON para Configuração | Facilidade de leitura por humanos e integração com scripts externos ou controlo de versões (Git). |
| Estrutura de Pastas Fixa | Garantir que o `AssetManager` e o `Compiler` saibam sempre onde procurar os ficheiros, simplificando a lógica do motor. |
| Auto-Open Last Project | Melhora a UX ao permitir que o utilizador continue o trabalho exatamente onde parou ao abrir o editor. |

---

## Critério de Aceitação

- [ ] Janela de Project Manager aparece ao iniciar o Caffeine Studio (se nenhum projeto for passado via CLI).
- [ ] Criar um projeto novo gera as pastas `assets/raw/`, `assets/processed/`, `scripts/` e `build/`.
- [ ] O ficheiro `project.caffeine` é criado com o nome e caminhos corretos.
- [ ] Selecionar um projeto na lista de "Recent Projects" carrega o editor com esse projeto.
- [ ] O botão "Open Project" abre um diálogo de seleção de ficheiro do sistema operativo.
- [ ] Erros de permissão de escrita ao criar projetos são reportados ao utilizador.
